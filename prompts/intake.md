# 边界与输入录入（Step 1）

## 用途

在开始创新点设计与交底书撰写前，用**少量问题**收敛案件边界。信息不全时可跳过，由 Agent 后续推断并**注明假设**。默认用户只需输入一段概念想法，不要求提供项目路径、代码或设计文档。

## 可选开场

```
为便于把这段想法设计成可专利化方案，先确认几项边界；可跳过，将按已有想法合理假设。
```

## 问题序列

### Q1：技术主题或产品模块

```
请用一句话描述这段想法所属技术主题、应用场景或产品模块，例如「多模态检索排序」「工业质检缺陷分级」。如果已经在上一条消息说明，可直接沿用。
```

### Q2：专利类型倾向

```
是否已有倾向的权利要求类型？方法 / 系统 / 装置 / 暂不确定
```

### Q3：文头联系人

```
交底书文头的技术联系人是否需要占位？需要则提供姓名/电话/邮箱；不需要则全部写「待填写」。
```

## 汇总

将已确认项用 3–6 行 bullet 复述给用户，再进入 `concept_design.md` 做概念拆解与创新方案设计。仅当用户提供项目路径、附件或已有材料时，再进入 `project_scan.md` 校准事实。
